## Issue Found During Validation

One failure was found during the first feature-test pass:

- `SportsCanonicalFlowTest::test_competition_summary_query`

Root cause:

- `GetCompetitionListSummary` called `withCount(...)` and then later called `select('competitions.*')`
- that `select(...)` overwrote the count subqueries generated by `withCount(...)`
- as a result, `total_provas` resolved as `null`, which surfaced as `0` in the test assertion

Fix applied:

- moved `select('competitions.*')` before `withCount(...)` in `app/Services/Desportivo/Queries/GetCompetitionListSummary.php`

Post-fix result:

- all Sports feature tests passed

## Recommended Release Checks

Before production rollout:

1. Run `php artisan migrate --force` against a staging clone of the target database.
2. Run `php artisan test tests/Feature/Sports` in the same environment.
3. Exercise the Desportivo screens with `VITE_SPORTS_USE_MOCK=false` to validate payload compatibility.
